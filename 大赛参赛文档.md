## 1. Demo 简介

### 是什么

**小呆相册（PhotoVault）** 是一款面向桌面的本地照片管理应用，基于 **Electron + React + TypeScript** 构建。它把照片扫描、时间线浏览、AI 语义搜索、地图可视化和智能去重整合在一个轻量级客户端里，所有数据默认存储在本地，无需联网即可使用。

### 面向谁

- 电脑里积累了大量照片、视频，苦于整理和查找的普通用户；
- 注重隐私，不希望把照片上传到云端的摄影爱好者；
- 需要快速去重、补全 GPS 和日期元数据的整理控。

### 主要功能

| 功能 | 说明 |
|------|------|
| **时间线浏览** | 按年月聚合照片，支持快速跳转到任意时间段，虚拟滚动流畅加载数万张照片。 |
| **AI 语义搜索** | 输入自然语言（如「海边日落」「猫咪」「生日聚会」），通过 CLIP 模型匹配照片内容，结果展示相似度分数。 |
| **智能去重** | 支持 MD5 精确去重 + 感知哈希相似去重，并智能推荐保留哪一张。 |
| **地图视图** | 自动读取照片 GPS 信息，在地图上聚合展示；支持单张 / 批量修改坐标。 |
| **元数据编辑** | 可编辑拍摄日期、补充 GPS 位置，支持多张照片批量修改坐标。 |
| **本地优先** | 使用 SQLite 存储索引，Sharp 生成缩略图，所有数据保存在用户指定目录，不上传云端。 |

## 2. Demo 创作思路

### 灵感来源

手机和相机里的照片越拍越多，但整理起来却越来越痛苦：

- 想找某张照片，翻了几十分钟文件夹也找不到；
- 同一场景拍了好多张，删掉重复照片很耗时；
- 很多老照片没有 GPS 信息，也记不清是在哪里拍的；
- 担心把照片传到云端会泄露隐私。

于是想做一款「本地、轻量、聪明」的照片管理工具，让用户在电脑上就能高效整理和回忆自己的照片。

### 想解决的问题

1. **照片太分散**：多个设备、多个文件夹，照片散落在各处，没有统一视图。
2. **查找困难**：只能靠文件夹名或日期回忆，无法用自然语言描述搜索。
3. **重复浪费空间**：相似照片和完全重复的照片占用大量磁盘空间。
4. **元数据缺失**：早期照片没有 GPS、拍摄日期错误，导致地图和时间线展示不完整。

### 为什么做这个方向

- **刚需明确**：几乎每个人都有照片管理的需求，市场空间大。
- **本地优先契合隐私趋势**：越来越多用户希望核心数据留在本地，不被云端分析。
- **AI 可以落地**：CLIP 等模型让「语义搜图」从概念变成普通用户也能用的功能。
- **参赛可行性高**：在 TRAE 协助下，一个人也能在较短时间内完成 Electron 桌面端 + AI 搜索的完整 Demo。

## 4. TRAE 实践过程

### 用 TRAE 完成 Demo 开发的完整流程

整个项目从 0 到可运行，基本是在 TRAE 的辅助下一步步推进的，大致流程如下：

1. **需求拆解与架构确认**
   - 我向 TRAE 描述目标：做一个本地照片管理桌面应用，需要照片扫描、浏览、去重、地图、AI 搜索。
   - TRAE 帮我确定了技术栈：Electron + React + Vite + TailwindCSS + SQLite + Leaflet + CLIP。

2. **基础框架搭建**
   - TRAE 生成了 Electron 主进程与渲染进程的 IPC 接口定义；
   - 搭建了照片库、浏览页、设置页等基础页面结构。

3. **核心功能实现**
   - **照片扫描与索引**：TRAE 写了文件遍历、EXIF 解析、SQLite 入库逻辑；
   - **时间线浏览**：按年月分组，结合 react-virtuoso 实现虚拟滚动；
   - **智能去重**：基于 MD5 和感知哈希实现精确/相似去重；
   - **地图视图**：读取 GPS 信息，在 Leaflet 上聚合展示，并支持批量改坐标；
   - **AI 语义搜索**：集成 @xenova/transformers 加载 CLIP 模型，生成文本和图像向量，按余弦相似度排序返回结果。

4. **细节打磨**
   - 地图抽屉支持鼠标滚轮横向滚动；
   - 调整 AI 搜索阈值、增加查询增强、前端显示相似度分数；
   - 重写 README，整理项目结构和构建方式。

5. **构建与打包**
   - 使用 `npm run build` 构建前后端；
   - 使用 `npm run dist` 打包出 NSIS 安装程序；
   - 使用 `npm run dist:portable` 打包便携版。
